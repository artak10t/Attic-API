Request Account Activation Token

REQUIRED PRIVILEGE - USER

Sends activation token to activate account. If USE_EMAIL_SUBSYSTEM is true in a config.php then uses email associated with session_id to send the activation token, otherwise sends activation token directly in data.

Request

Only 'POST' method is accepted


TAKES

Field Mandatory Default Type Constraints Description
session_id Yes No Default string Session id
prepend No "" string Text/Link that will be prepended before the token. Only prepends when USE_EMAIL_SUBSYSTEM is true

EXAMPLE

{
  "session_id": "942A5659AD83851E54BC8D818764BE9D126F25E510EC9C233AE57F25D757BBD4"
}

RESPONSE

Returns after request

Error Code Message Data Description
0 Success session_id Session token
-1 E_NOT_LOGGED_IN Invalid session
-4 E_DISABLED Account is disabled
-50 E_ONLY_POST Only POST method allowed
-51 E_FIELD_NOT_SET One or more mandatory fields are not set
-52 E_FIELD_INVALID One of the fiels is not valid
-53 E_MAIL_FAILED Mail sending failed

EXAMPLE

{
  "code": 0,
  "msg": "",
  "data": {
    "token": "41BE526B1853559BF7AFF953FF035FEB6EBFAFC92668306155D8E16E6581AB4F"
  }
}
